Quantum Cryptography

In this activity, we will first simulate a quantum key exchange. Then we will use Binary Cipher to exchange messages.

The sender first needs to communicate a string of bits to the receiver to be used as a key. These are sent as photons. A 1 can be represented by either ↕ or ⤢ (chosen randomly) and a 0 can be represented by either ↔ or ⤡ (chosen randomly). The receiver randomly chooses a detector for each photon. For ↔ or ↕ the correct detector is 〸, and for ⤡ and ⤢ the correct detector is X. The receiver then tells the sender which detectors they used. The sender replies with the list of correct detectors. They then put together the bits corresponding to these photons for their key.

Group Message Exchange
  1. The Senders access the appropriate Google Sheet above, and make a set of random choices for photons to send.
  2. The Receivers access the appropriate Google Sheet above, and make a set of random choices for detectors to use.
  3. After the both the Senders and Receivers make their choices, the Senders will get a list of the detector choices made by the Receivers. They decide which ones are guaranteed to correctly detect the photon. So for ↔ or ↕ the correct detector is 〸 and for ⤡ and ⤢ the correct detector is X. For these dectors, the Expected bits will match the Received bit.
  4. In the Quantum Message forum in the Student Messages section of Blackboard, the Senders post the list of correct detectors. So they will post something like: 1, 4, 5, 7, 8, 12, 14, 16. (Do not post the bits.)
  5. The key is the list of bits corresponding to correct detector positions. Both the Sender and Receiver (and only the Sender and Receiver) know what these bits are.
  6. Both the Sender and Receiver should create messages using the Binary Cipher. You can access the necessary tools in the Tools section or on the Binary Cipher activity page. Post cipher texts under the list of correct detectors in the Quantum Message forum, and decrypt the message from the other party.
